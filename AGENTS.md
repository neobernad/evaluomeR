This file gives AI agents and new contributors the orientation needed to work

## Package Overview

**evaluomeR** (v1.8.1) is a Bioconductor R package that evaluates the
reliability of bioinformatics metrics by analysing the stability and goodness
of classifications produced by those metrics. The three core analyses are:

- **Correlation** — Pearson correlation matrix across metrics
- **Stability** — Bootstrap Jaccard stability indices over a range of k
- **Quality** — Silhouette and Calinski-Harabasz quality indices over a range of k

An advanced pipeline called **ATSC** (Automated Trimmed & Sparse Clustering)
combines all three analyses with RSKC (Robust Sparse k-Means Clustering) to
automatically select the optimal number of clusters and down-weight irrelevant
metrics.

### Install

```r
# Bioconductor (stable)
BiocManager::install("evaluomeR")

# GitHub (development)
devtools::install_github("neobernad/evaluomeR")

# Conda
# conda install -c bioconda bioconductor-evaluomer
```

- Requires R >= 3.6
- License: GPL-3
- Bug reports: https://github.com/neobernad/evaluomeR/issues
- Docs site: https://neobernad.github.io/evaluomeR/

## Input Data Contract

### Expected input shape

All analysis functions accept either a `data.frame` or a `SummarizedExperiment`.

```
           Description  Metric_A  Metric_B  Metric_C ...
row 1      "Sample_1"   0.82      1.14      0.67
row 2      "Sample_2"   0.91      0.98      0.74
...
```

- The **first column** must be named `Description` and contain instance/sample IDs.
- All remaining columns are numeric metric measurements.
- Helper `createSE()` in `R/helpers.R` wraps a data frame into a `SummarizedExperiment`.

### Bundled datasets

| Dataset | Description |
|---------|-------------|
| `rnaMetrics` | RNA quality metrics (RIN, DegFact) for 16 aliquots — simplest test dataset |
| `ontMetrics` | 19 structural ontology metrics (OBO Foundry + AgroPortal) |
| `bioMetrics` | Biological pathway metrics (reachability, efficiency) |

Load with `data("rnaMetrics")`, `data("ontMetrics")`, or `data("bioMetrics")`.

## Supported Clustering Backend Interfaces (CBIs)

Defined in `pkg.env$cbi` inside `R/helpers.R`:

```
"kmeans"    "clara"    "clara_pam"    "hclust"    "pamk"    "pamk_pam"    "rskc"
```

Pass as the `cbi` argument to `quality()`, `qualityRange()`, `ATSC()`, etc.
Default is `"kmeans"` for most functions and `"clara"` for ATSC.

## Running Tests

There is **no testthat / RUnit harness**. Tests are standalone R scripts in
`tests/`. Run them by sourcing from an R session with the package installed:

```r
library(evaluomeR)
source("tests/testAll.R")          # end-to-end smoke test
source("tests/testStability.R")    # stability index tests
source("tests/testQuality.R")      # quality index tests
source("tests/testATSC.R")         # ATSC pipeline tests
# ... etc. — see tests/ for the full list of 18 scripts
```

Some scripts reference external datasets (`golub`, `nci60_k8`) that require
separate Bioconductor data packages or local `.rda` files not committed to
this repo.

The canonical test run used by CI is `R CMD check`, which exercises the
package examples and vignette code (not the `tests/` scripts directly).

## Coding Conventions

- **Documentation**: all exported functions use roxygen2 (`@param`, `@return`,
  `@examples`, `@export`). Run `devtools::document()` to regenerate `man/` and
  `NAMESPACE`. Never edit `.Rd` files or `NAMESPACE` by hand.
- **Function naming**: exported functions use camelCase (e.g. `stabilityRange`,
  `getOptimalKValue`). Internal helpers use lower_snake or camelCase
  inconsistently — follow the style of the file you are editing.
- **I/O pattern**: functions accept either a plain `data.frame` or a
  `SummarizedExperiment`; they normalise internally and always return a
  `SummarizedExperiment` (or `MultiAssayExperiment` / named list of SEs for
  `*Range` / `*Set` variants).
- **Package-scoped state**: mutable package state lives in `pkg.env` (a
  `new.env()`), defined in `R/helpers.R`. Avoid introducing global variables
  outside this environment.
- **Parallel execution**: functions that support parallelism accept a
  `numCores` argument and use `parallel::mclapply` internally.
- **Seed handling**: a default seed (`13606`) is stored in `pkg.env$seed`.
  Functions accept an optional `seed` argument to override it for
  reproducibility.
- **Adding a new CBI**: register the name in `pkg.env$cbi` (`R/helpers.R`),
  implement a CBI function in `R/internalClusterboot.R`, and wire it into the
  dispatch logic in `R/internalFunctions.R`.

## Agent Workflow

### Ponytail (optional, local install)

[Ponytail](https://github.com/DietrichGebert/ponytail) lazy-senior-dev mode is
**not committed to this repo** — agent tooling is a per-developer choice. To
enable it in Cursor locally:

```bash
mkdir -p .cursor/rules
curl -fsSL https://raw.githubusercontent.com/DietrichGebert/ponytail/main/.cursor/rules/ponytail.mdc \
  -o .cursor/rules/ponytail.mdc
```

Default intensity: **full**. Say `stop ponytail` / `normal mode` to turn it off
for a session. `.cursor/` is gitignored.

### Area-specific tests (required after code changes)

There is no testthat harness. After editing package code, run the matching
script(s) from an R session with `evaluomeR` installed:

```r
library(evaluomeR)
source("tests/<script>.R")
```

| Area changed | R source | Test script(s) |
|--------------|----------|----------------|
| Stability | `R/stabilityIndex.R`, `R/internalFunctions.R` (bootstrap/Jaccard) | `tests/testStability.R` |
| Quality | `R/qualityIndices.R` | `tests/testQuality.R`, `tests/testCH_index.R` |
| Correlation / metrics plots | `R/correlation.R`, `R/metricsAnalysis.R` (plotting only) | `tests/testAnalysis.R`, `tests/testMetricsRelevancy.R` |
| ATSC pipeline | `R/metricsAnalysis.R` (ATSC, RSKC helpers) | `tests/testATSC.R`, `tests/testATSC_GoldStandard.R`, `tests/testL1_clustering.R` |
| Preprocessing / PCA | `R/dataPreprocessing.R`, `R/predictions.R` | `tests/testDataPreprocessing.R`, `tests/testPCA.R` |
| Clustering backends (CBI) | `R/internalClusterboot.R`, `R/helpers.R` (`pkg.env$cbi`) | `tests/testCBI.R`, `tests/testARI.R` |
| Helpers / package-wide | `R/helpers.R`, `R/data.R`, `DESCRIPTION`, `NAMESPACE` | `tests/testAll.R` |
| CI / docs only | `.github/`, `man/`, `vignettes/` | `R CMD check` (no `tests/` scripts) |

Parallel benchmark scripts (`tests/*_parallel.R`) are excluded from the package
build; run them only when changing parallel execution paths.

When a change spans multiple areas, run every matching script before claiming
the work is done.
